## 一、核心原则（铁律）

### 1.1 三大铁律

| 铁律 | 说明 | 实现方式 |
|------|------|---------|
| **严禁鼠标键盘** | 禁止 pyautogui、SendInput、mouse_event、PostMessage 模拟 | 用 UIA `InvokePattern.Invoke()` |
| **严禁影响其他应用** | 所有操作必须精确锁定目标窗口及其子控件 | `win32gui.EnumWindows` 按精确标题查找 |
| **通过控件 Name 精确调用** | 按钮通过文本内容定位，不靠坐标 | `ButtonControl(Name=button_name)` |

### 1.2 技术栈

```
pywin32               # win32gui 窗口查找 + PrintWindow 截图
Pillow                # ImageGrab 截屏区域 + 图像处理
```

## 二、源码侧需要提供什么（关键！）

截图脚本能否工作，**80% 取决于源码是否预留了截图接口**。以下是源码必须提供的内容。

### 2.1 按钮控件命名（AutomationProperties）

WPF 按钮的 `Content` 文本会自动成为 UIA 的 Name 属性，**无需额外设置**。但建议显式设置以确保稳定：

```xml
<!-- ✓ 推荐：显式设置 AutomationProperties.Name -->
<Button x:Name="BtnRefreshOverview"
        AutomationProperties.Name="刷新概览"
        Content="⟳ 刷新概览"
        Click="BtnRefreshOverview_Click"/>

<!-- ✓ 最简：Content 文本自动成为 UIA Name -->
<Button x:Name="BtnForecast" Content="▶ 运行负荷预测" Click="BtnForecast_Click"/>
```

**注意事项**：
- 按钮文本包含图标符号（如 `⟳ 刷新概览`）时，UIA Name 也包含该符号，截图脚本传入 `button_name` 时必须带图标
- 同一模块内不应有重名按钮（包括图标）
- 跨模块可以重名（截图脚本只在当前模块作用域内查找）

### 2.2 导航项支持 InvokePattern（关键！）

**问题**：WPF `Border` 默认不支持 UIA `InvokePattern`，截图脚本无法通过 UIA 点击导航项。

**解决方案**：把导航项从 `Border` 改为 `Button`，Button 天然支持 `InvokePattern`。

```csharp
// ✓ 正确：导航项用 Button
private void AddNavItem(int moduleId, string icon, string name, string nameEn, bool isHome)
{
    var btn = new Button
    {
        Padding = new Thickness(14, 8, 8, 8),
        Margin = new Thickness(0, 1, 0, 0),
        Cursor = Cursors.Hand,
        Tag = moduleId,
        Background = Brushes.Transparent,
        BorderThickness = new Thickness(0),
        HorizontalContentAlignment = HorizontalAlignment.Stretch,
        Name = $"Nav_{moduleId}"
    };

    // 关键：设置 UIA 属性
    System.Windows.Automation.AutomationProperties.SetName(btn, name);
    System.Windows.Automation.AutomationProperties.SetAutomationId(btn, $"NavBtn_{moduleId}");

    // 内容（保持原 Border 的视觉效果）
    var stack = new StackPanel { Orientation = Orientation.Horizontal };
    stack.Children.Add(new TextBlock { Text = icon, FontSize = 16 });
    stack.Children.Add(new TextBlock { Text = name, FontSize = 12 });
    btn.Content = stack;

    // 点击事件
    btn.Click += (s, e) => OpenModule(moduleId);

    NavPanel.Children.Add(btn);
}
```

**各语言等价方案**：

| 语言 | 控件 | 设置方式 |
|------|------|---------|
| C# WPF | Button | `AutomationProperties.SetName(btn, name)` |
| Python PyQt6 | QPushButton | `button.setObjectName('btn_xxx')` |
| C++ Qt6 | QPushButton | `button->setObjectName('btn_xxx')` |
| PHP Web | `<button>` | `id="btn-xxx"` |
| C GTK3 | GtkButton | `gtk_widget_set_name(btn, "btn-xxx")` |
| Rust egui | Button | `ui.add(Button::new("text").id_source("btn_xxx"))` |

### 2.4 模态弹窗设计（关键！）

**问题**：WPF `Window.ShowDialog()` 是模态的，会阻塞 UI 线程。UIA `Invoke()` 调用按钮 Click 事件时，如果 Click 中调用 `ShowDialog()`，`Invoke()` 会阻塞等待 `ShowDialog()` 返回——死锁！

**源码侧无需特殊处理**，但弹窗必须满足以下要求以便截图脚本关闭：

```csharp
// ✓ 弹窗底部必须有"确定"按钮
public void ShowResultDialog(string title, string summary, string details)
{
    var dlg = new Window
    {
        Title = title,
        Width = 480,
        Height = 360,
        WindowStartupLocation = WindowStartupLocation.CenterOwner,
        Owner = this,
        // ...
    };

    var okBtn = new Button
    {
        Content = "确定",  // 关键：按钮文本必须是"确定"
        Width = 100,
        Height = 36,
        // ...
    };
    okBtn.Click += (s, e) => dlg.Close();

    dlg.ShowDialog();  // 模态显示
}
```

**弹窗关闭按钮文本约定**（按优先级）：
1. `确定`
2. `关闭`
3. `×`
4. `OK`
5. `Close`
6. `取消`

### 3.2 Action 类型

| Action | 说明 | 必需字段 | 可选字段 |
|--------|------|---------|---------|
| `screenshot` | 截取主窗口 | `screenshot_name` | - |
| `screenshot_with_popup` | 截取主窗口+弹窗 | `screenshot_name` | - |
| `login` | 点击登录按钮 | `button_name` | - |
| `navigate` | 导航到模块 | `module_name` | - |
| `click_button` | 点击按钮 | `button_name` | `automation_id`, `scope` |
| `click_button_with_popup` | 点击按钮+截图弹窗+关闭 | `button_name` | `automation_id`, `scope` |
| `scroll` | 滚动展示 | `scroll_count` | - |
| `wait` | 等待 | `seconds` | - |

### 3.3 scope 字段说明

| scope | 说明 | 使用场景 |
|-------|------|---------|
| `window` (默认) | 在整个主窗口查找按钮 | 顶部工具栏按钮、底部日志"清空"按钮 |
| `content` | 只在当前模块内容区查找按钮 | 模块内的"清空"按钮（避免与日志区"清空"重名冲突） |

### 3.4 步骤覆盖铁律

- **全量覆盖**：每个模块的每个按钮都必须有独立 step
- **首屏截图**：每个模块必须有 `navigate` + `screenshot`
- **滚动展示**：每个模块必须有 `scroll` step
- **弹窗操作**：所有触发弹窗的按钮必须用 `click_button_with_popup`
- **严禁省略**：demo_steps.json 中不得用"等"省略按钮

### 4.1 整体流程


### 4.2 关键函数

#### 4.2.1 查找主窗口

```python
def find_main_window():
    """精确查找主窗口，绝不动其他窗口"""
    result = []
    def callback(hwnd, _):
        if win32gui.IsWindowVisible(hwnd):
            title = win32gui.GetWindowText(hwnd)
            if title == WINDOW_TITLE:
                result.append(hwnd)
    win32gui.EnumWindows(callback, None)
    # 如果有多个同名窗口（登录窗+主窗），选面积最大的
    if len(result) > 1:
        result.sort(key=lambda h: win32gui.GetWindowRect(h)[2] * win32gui.GetWindowRect(h)[3], reverse=True)
    return result[0] if result else None
```

#### 4.2.2 截图（PrintWindow）

```python
def capture_printwindow(hwnd, filename):
    """通过 PrintWindow 截取指定窗口"""
    rect = win32gui.GetWindowRect(hwnd)
    w = rect[2] - rect[0]
    h = rect[3] - rect[1]
    hwndDC = win32gui.GetWindowDC(hwnd)
    mfcDC = win32ui.CreateDCFromHandle(hwndDC)
    saveDC = mfcDC.CreateCompatibleDC()
    bitmap = win32ui.CreateBitmap()
    bitmap.CreateCompatibleBitmap(mfcDC, w, h)
    saveDC.SelectObject(bitmap)
    # PW_RENDERFULLCONTENT = 2，确保 WPF 内容完整渲染
    windll.user32.PrintWindow(hwnd, saveDC.GetSafeHdc(), 2)
    bitmap.SaveAsFile(filename, win32con.BITMAPFILETYPE_PNG)
    saveDC.DeleteDC()
    mfcDC.DeleteDC()
    win32gui.ReleaseDC(hwnd, hwndDC)
    win32gui.DeleteObject(bitmap.GetHandle())
```

#### 4.2.4 查找并点击按钮（核心）

```python
def find_button(main_window, button_name, automation_id=None,
                timeout=3, scope="window", main_hwnd=None):
    """通过 UIA 按按钮 Name 精确查找"""
    if scope == "content":
        # 只在当前模块内容区查找（避免重名冲突）
        search_root = find_content_area(main_window)
    else:
        search_root = main_window

    # 方式1：按 Name 精确匹配
    btn = search_root.ButtonControl(Name=button_name, searchDepth=10)
    if btn.Exists(timeout, 0.5):
        return btn

    # 方式2：按 AutomationId 查找
    if automation_id:
        btn = search_root.ButtonControl(AutomationId=automation_id, searchDepth=10)
        if btn.Exists(1, 0.5):
            return btn

    return None

def invoke_control(control):
    """调用 InvokePattern.Invoke()"""
    invoke = control.GetInvokePattern()
    if invoke:
        invoke.Invoke()
        return True
    return False
```

#### 4.2.5 关闭弹窗

```python
def detect_popup(main_hwnd, timeout=1.0):
    """检测是否有模态弹窗"""
    popups = []
    def callback(hwnd, _):
        if hwnd != main_hwnd and win32gui.IsWindowVisible(hwnd):
            if win32gui.IsWindowEnabled(hwnd):
                # 检查是否是主窗口的子窗口或弹窗
                parent = win32gui.GetParent(hwnd)
                if parent == main_hwnd or win32gui.GetWindowText(hwnd):
                    popups.append(hwnd)
    win32gui.EnumWindows(callback, None)
    return (True, popups[0]) if popups else (False, None)

def close_popup(popup_hwnd):
    """通过 UIA 查找弹窗中的"确定"按钮并 Invoke"""
    popup_element = auto.ControlFromHandle(popup_hwnd)
    if not popup_element:
        return False
    for name in ["确定", "关闭", "×", "OK", "Close", "取消"]:
        btn = popup_element.ButtonControl(Name=name, searchDepth=10)
        if btn.Exists(1, 0.5):
            try:
                invoke = btn.GetInvokePattern()
                if invoke:
                    invoke.Invoke()
                    return True
            except:
                pass
    return False
```

### 4.3 模态弹窗死锁解决方案（关键！）

**问题**：UIA `Invoke()` 调用按钮 Click，Click 中调用 `ShowDialog()`，`ShowDialog()` 阻塞等待用户关闭，`Invoke()` 阻塞等待 Click 返回——死锁！

**解决方案**：把 `Invoke()` 放到子线程中执行，主线程等待弹窗出现后截图再关闭。

```python
import threading

def click_button_with_popup(main_window, button_name, filename):
    # 1. 找到按钮
    btn = find_button(main_window, button_name)
    if btn is None:
        print(f"  [按钮警告] 未找到: {button_name}")
        return

    # 2. 子线程执行 Invoke（避免模态死锁）
    invoke_result = {"ok": False}
    def do_invoke():
        try:
            invoke_result["ok"] = invoke_control(btn)
        except Exception as e:
            print(f"  [Invoke 异常] {e}")

    t = threading.Thread(target=do_invoke, daemon=True)
    t.start()
    t.join(timeout=0.8)  # 等待 Invoke 触发（不等返回）

    if invoke_result["ok"]:
        print(f"  [按钮] 已 Invoke: {button_name}")
    elif t.is_alive():
        print(f"  [按钮] 已触发(等待弹窗): {button_name}")

    # 3. 等待弹窗出现
    time.sleep(1.5)

    # 4. 截图（含弹窗）
    capture_with_popup(main_hwnd, filename)

    # 5. 关闭弹窗
    _, popup = detect_popup(main_hwnd, timeout=1.0)
    if popup:
        if close_popup(popup):
            print("  [弹窗] 已关闭")

    # 6. 等待 Invoke 子线程返回（弹窗关闭后 ShowDialog 返回）
    if t.is_alive():
        t.join(timeout=2.0)
```

## 五、需要注意的关键问题

### 5.1 运行时资源查找崩溃

**问题**：`Application.Current.FindResource("ExcellentGrad")` 在应用级查找资源，但资源定义在 UserControl.Resources 中（模块级），导致 `ResourceReferenceKeyNotFoundException` 崩溃。

**解决方案**：避免在数据绑定时用 `FindResource`，改用直接的颜色值。

```csharp
// ✗ 错误：应用级查找模块级资源
public Brush GradeBrush => (Brush)Application.Current.FindResource(GradeBrushKey);

// ✓ 正确：直接用颜色值创建画刷
public Brush GradeBrush
{
    get
    {
        try { return (Brush)new BrushConverter().ConvertFromString(SohColorHex); }
        catch { return Brushes.Gray; }
    }
}
```

### 5.2 OxyPlot 图表 API 兼容性

**问题**：OxyPlot 2.1.2 中 `BarSeries` 要求 Y 轴是 `CategoryAxis`，但代码用 `LinearAxis`，导致 `BarSeries requires a CategoryAxis on the Y Axis` 异常。

**解决方案**：改用 `LineSeries`，与现有 `LinearAxis` 兼容。

```csharp
// ✗ 错误：BarSeries 需要 CategoryAxis
var series = new BarSeries { ... };
series.Items.Add(new BarItem { Value = x });

// ✓ 正确：LineSeries 兼容 LinearAxis
var series = new LineSeries { ... };
series.Points.Add(new DataPoint(h, x));
```

### 5.3 XAML 资源路径问题

**问题**：`Source="Styles/Colors.xaml"` 在 Styles 目录下的 Controls.xaml 中，路径会解析为 `Styles/Styles/Colors.xaml`，导致 `XamlParseException` 运行时崩溃。

**解决方案**：路径相对于当前 XAML 文件位置。

```xml
<!-- Styles/Controls.xaml 中 -->
<!-- ✗ 错误：路径会变成 Styles/Styles/Colors.xaml -->
<ResourceDictionary Source="Styles/Colors.xaml"/>

<!-- ✓ 正确：相对于当前文件 -->
<ResourceDictionary Source="Colors.xaml"/>
```

### 5.5 截图空白问题

**问题**：`PrintWindow` 截图有时全黑或全白（WPF 渲染时机问题）。

**解决方案**：检测截图是否为空白，空白时用 `ImageGrab.grab` 兜底。

```python
def capture_printwindow(hwnd, filename):
    # ... PrintWindow 截图 ...
    # 检测空白
    img = Image.open(filename)
    small = img.resize((50, 30))
    pixels = list(small.getdata())
    if len(set(pixels)) < 5:  # 颜色种类太少，可能是空白
        # 用 ImageGrab 兜底
        rect = win32gui.GetWindowRect(hwnd)
        ImageGrab.grab(bbox=rect).save(filename)
```

### 5.6 弹窗未检测到

**问题**：Toast 提示（非模态）不会被 `detect_popup` 检测到，因为 Toast 不阻塞主窗口。

**解决方案**：Toast 不需要关闭，会自动消失。日志中输出"可能为 Toast"即可。

### 5.7 窗口大小

**铁律**：窗口大小设置为桌面分辨率的 3/4，严禁全屏截图。

```python
window_w = int(screen_w * 0.75)
```

## 六、质量验收标准

### 6.1 截图验收

| 检查项 | 要求 | 验证方式 |
|--------|------|---------|

- 每个步骤都有对应日志输出
- 无"未找到按钮"警告（或仅有少量且已确认原因）
